Link State Packet

OSPF uses link state packets (LSPs) which are special datagrams that determine the names of and the cost or distance to any neighbouring routers and associated networks. The diagram below shows the information that each field of an OSPF packet contains, click on a field in the packet for more information:

Version Number

Size 1 byte

Identifies the OSPF version used.

Note: the lengths in the diagram are in Bytes (8 Bits = 1 Byte)

Type

Size 1 byte

Identifies the OSPF packets type

Note: the lengths in the diagram are in Bytes (8 Bits = 1 Byte)

Packet Length

Size 2 bytes

Specifies the packet length, including the OSPF header, in bytes.

Note: the lengths in the diagram are in Bytes (8 Bits = 1 Byte)

Router ID

Size 4 bytes

Identifies the source of the packet.

Note: the lengths in the diagram are in Bytes (8 Bits = 1 Byte)

Area ID

Size 4 bytes

Identifies the area to which the packet belongs. All OSPF packets are associated with a single area.

Note: the lengths in the diagram are in Bytes (8 Bits = 1 Byte)

Check Sum

Size 2 bytes

Checksum error is used to identify errors in the packet.

Note: the lengths in the diagram are in Bytes (8 Bits = 1 Byte)

Authentication Type

Size 2 bytes

All OSPF protocol exchanges are authenticated. The authentication type is configurable on a per-area basis.

Note: the lengths in the diagram are in Bytes (8 Bits = 1 Byte)

Authentication

Size 8 bytes

Contains authentication information.

Note: the lengths in the diagram are in Bytes (8 Bits = 1 Byte)

Data

Size Variable

Contains encapsulated upper-layer information. The size of this section is variable.

Note: the lengths in the diagram are in Bytes (8 Bits = 1 Byte)
